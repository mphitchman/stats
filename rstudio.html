Tips for Getting Started with RStudio in MATH 140

TIP 1: Install R and RStudio. Read Section 1.1.1 of the text ModernDive for instructions on installing them both.

TIP 2: Organize a folder system on your computer

  1. Create a folder dedicated to this class on your computer, perhaps the desktop, with a descriptive name, say “math140”.

  2. Create a subfolder in your “math140” folder for each RStudio project you work on. For instance, you might create a folder called “lab1” in your “math140” folder ahead of our first lab.

TIP 3: Create RStduio projects. Project files in Rstudio give you a good way to organize your work in RStudio. You will probably have at least 5 projects before the end of the semester, one for each of our labs, and one for your term project. To create a project, follow these steps:

  1. Open RStudio

  2. Create a project from the File menu by moving the cursor to the upper-right corner of your window and selecting Project (None)>New Project

  1. Select Existing Directory in the pop-up menu:

  2. Click Browse and navigate to where you created the lab1 folder. Select this folder and then click Create Project.

  • Your R project has now been created. Note that in the upper right-hand corner, the program indicates that you are working on the project named lab1. In the future you can click on this spot in the upper right corner to change the project you want to work on or create a new one.

  • Note also that if you click on the Files tab in the lower right pane in RStudio, you can access the file system on your computer, and RStudio is showing the contents of your current project directory.

  • My mature project folders often contain three types of files:

    • R script files (discussed below)
    • data files (usually .csv files)
    • output files (pdf files, images of graphs, webpages)

TIP 4: Create scripts in RStudio. Think of a script as a place to write down commands you want to use in a project. You can also type comments (begin a line with a pound sign #) to help someone understand the commands that follow (including yourself two weeks later!). To create a script, go to the upper left corner of your window and click on the green + symbol. Then select R script.

As already mentioned, scripts are a place to write down commands you want to use in your work. To actually execute lines of code in your script, place your cursor anywhere on that line of code and click Run.

Alternatively, you can use a keyboard shortcut:

  • Command + return on a Mac
  • ctrl + enter on a PC

TIP 5: Reading a local file into RStudio.

To import a .csv file that is saved locally on your computer, use the command read.csv(path to file/filename)

The ‘path to file’ part of this command is necessary if the file you want is not in your RStudio working directory. You can set your working directory to a particular folder by going to the Files tab in the lower right pane, navigating to the folder you want, then clicking on More and selecting “Set as Working Directory.”

It’s simplest for me to save local data files in the same folder as the scripts I write to analyze them, and then in RStudio make that folder the working directory when I’m working on them.

For instance, suppose in a folder called “awesome” I have a script file called legos.R, and I also have a data file called legosets.csv. If I set my working directory in RStudio to the awesome folder, then the following line in my legos.R script will read the data file into my RStudio session, and give it the working name ‘legos’:

legos = read.csv("legosets.csv")

TIP 6: Reading a csv files into RStudio from the web.

To import a .csv file directly from the web into RStudio, such as one of the ones below, we use the following command:

read.csv("web address")

For instance, to import the governors data file, and give it the name gov, we enter the following at the console prompt in RStudio (you can copy and paste!):

gov = read.csv("https://mphitchman.com/stats/data/governors.csv")

This works regardless of your working directory :).

TIP 7: Installing and Using Packages.

RStudio comes with a bunch of built-in functions, but packages provide more! Here is a link to a very nice introduction to packages in ModernDive. I suggest you read all of Section 1.3.

To install a package, follow the steps in the link above.

Once a package is installed in your copy of RStudio, you can load it into a current session with the library() command. For instance, the ggplot2 package, used for data visualization, can be loaded into a session by executing this command:

library("ggplot2")

If you get an error when you run this, it means ggplot2 hasn’t been installed on your computer.

Note: You only need to install the packages once, but you will need to load the packages each time you use RStudio.

TIP 8: R Markdown.

Your lab reports will be written using R Markdown, and perhaps parts of your term project. R Markdown files allow you to knit together text, R code, and images into a fine looking report. More on this later.
